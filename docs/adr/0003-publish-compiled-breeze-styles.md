# Publish compiled Breeze styles

Breeze UI will use Tailwind internally and publish two compiled stylesheet entry points. Every application imports `styles.css`, which contains fonts, semantic tokens, component-scoped normalization, and component styles; greenfield applications may additionally import `reset.css` first to opt into Tailwind Preflight and Breeze's global document baseline. Consumers do not scan Breeze sources or align their Tailwind version with Breeze; semantic CSS variables and cascade layers provide the stable styling boundary. Components expose `className` on their rendered parts for placement and composition, while supported visual differences use typed Breeze variants; unstyled modes, arbitrary visual-property props, and internal class names are not public customization contracts.
