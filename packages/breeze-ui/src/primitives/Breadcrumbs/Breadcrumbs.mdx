# Breadcrumbs

`Breadcrumbs` shows a user's position in a hierarchy and provides links back to its ancestors.

## Import

```tsx
import { Breadcrumbs } from '@motech-development/breeze-ui';
```

## When to use

Use breadcrumbs for pages nested several levels deep when the hierarchy helps users orient themselves. Breadcrumbs supplement rather than replace primary navigation.

## When not to use

Prefer `NavigationList` for persistent destinations, `Tabs` for switching peer views, and a simple `Link` back to the previous step for a short linear task.

## Basic example

```tsx
import { Breadcrumbs } from '@motech-development/breeze-ui';

<Breadcrumbs.Root aria-label="Breadcrumb">
  <Breadcrumbs.Item id="home" href="/">
    Home
  </Breadcrumbs.Item>
  <Breadcrumbs.Item id="settings" href="/settings">
    Settings
  </Breadcrumbs.Item>
  <Breadcrumbs.Item id="profile">Profile</Breadcrumbs.Item>
</Breadcrumbs.Root>;
```

## Anatomy and routing

| Part               | Purpose                                                                   |
| ------------------ | ------------------------------------------------------------------------- |
| `Breadcrumbs.Root` | Renders the semantic ordered trail and applies group-wide disabled state. |
| `Breadcrumbs.Item` | Renders one keyed ancestor link or the non-link current location.         |

Give `Root` an accessible name such as `aria-label="Breadcrumb"`. Every `Item` requires a stable `id: string | number`. An item with `href` is a link; omit `href` only from the final current item, which receives `aria-current="page"`.

Local `/path` links use the optional `BreezeProvider` router adapter. External URLs and native anchor behaviour remain browser-owned. `disabled` on `Root` disables navigable crumbs; `disabled` on an item disables only that link. Breadcrumbs have no controlled, read-only, loading, invalid, empty, or error state. Do not render an empty trail.

## Native attributes and styling

Relevant ordered-list and list-item attributes, including `aria-*` and `data-*`, are supported. Inline `style` and conflicting collection identifiers are intentionally excluded.

## Accessibility, internationalisation, and limitations

Links use ordinary Tab navigation and Enter activation. Keep labels short but meaningful outside the trail, preserve hierarchy order, and never make the current item a link to itself. Long translated labels wrap; do not truncate the only identifying text. Separators are visual and generated by Breeze. Logical layout follows the provider direction.

Common mistakes include omitting the root label, supplying `href` on the current page, using array positions as persistent ids, and using breadcrumbs for workflow progress. Related components: use `Stepper` for progress, `NavigationList` for primary navigation, and `Link` for a standalone route.
