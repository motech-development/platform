# Calendar

`Calendar` presents an always-visible, keyboard-navigable month grid for selecting one calendar date.

## Import

```tsx
import { Calendar } from '@motech-development/breeze-ui';
```

## When to use

Use `Calendar` when the month grid should remain visible or is central to the task.

## When not to use

Use `DatePicker` for a compact labelled form control, `DateField` for direct segmented entry, and `RangeCalendar` for an inclusive start and end date. A calendar is not a replacement for a labelled form field when space or form conventions call for one.

## Basic example

```tsx
import { Calendar } from '@motech-development/breeze-ui';

<Calendar.Root defaultValue="2026-07-20">
  <Calendar.Header>
    <Calendar.PreviousButton />
    <Calendar.Heading />
    <Calendar.NextButton />
  </Calendar.Header>
  <Calendar.Grid />
</Calendar.Root>;
```

## Anatomy

| Part                      | Purpose                                                             |
| ------------------------- | ------------------------------------------------------------------- |
| `Calendar.Root`           | Owns selection, visible month, constraints and calendar semantics.  |
| `Calendar.Header`         | Lays out the navigation controls and current-month heading.         |
| `Calendar.PreviousButton` | Moves to the previous calendar page.                                |
| `Calendar.Heading`        | Supplies the locale-formatted visible month heading and grid label. |
| `Calendar.NextButton`     | Moves to the next calendar page.                                    |
| `Calendar.Grid`           | Renders weekday headings and the complete date-cell grid.           |

Keep the navigation parts in `Header` and `Grid` inside `Root`. Breeze owns cells, selected styling and unavailable-date semantics; applications do not render individual days.

## Selection and constraints

The public value is a `YYYY-MM-DD` string such as `'2026-07-20'`, or `null` when empty. Use `defaultValue` for uncontrolled selection, `value` with `onChange` for controlled selection, or `value` with `readOnly` for immutable selection. `onChange` receives the newly selected stable date or `null`.

`minValue` and `maxValue` bound selection. `isDateUnavailable(date)` receives the same stable format and disables a specific date when it returns `true`. `invalid` announces that the current selection is unacceptable but does not render an error message; place application-owned explanatory text next to the calendar. `disabled` prevents focus and selection. `visibleMonths` displays adjacent months but does not change the single-date value model. There is no loading state.

## Keyboard, labelling and locale

- Arrow keys move by day or week according to locale direction. Home and End move within a week; Page Up and Page Down move by month, with the platform modifier moving by a larger period.
- Enter or Space selects the focused available date. The previous and next buttons are separately tabbable and operable with Enter or Space.
- `Heading` gives the grid its accessible visible-period label. Keep it in the composition even if surrounding text also describes the calendar.
- Weekday names, week boundaries, date announcements and month headings follow the `BreezeProvider` locale. Logical navigation and arrow icons adapt to right-to-left direction.
- `weekdayStyle="short"` is the default; use `narrow` only when space is genuinely constrained because one-letter labels can be ambiguous.

## Common mistakes and related components

Do not pass date objects or locale-formatted strings, render your own day buttons inside `Grid`, or hide unavailable dates instead of disabling them. Avoid large `visibleMonths` values in narrow layouts. Use `RangeCalendar` for a period, `DatePicker` for a compact popup, and `FormattedDateTime` for read-only prose.
