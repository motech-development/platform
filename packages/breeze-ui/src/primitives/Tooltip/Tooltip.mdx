# Tooltip

`Tooltip` gives an existing trigger concise, non-essential descriptive help without adding an action surface.

## Import

```tsx
import { Tooltip } from '@motech-development/breeze-ui';
import { WarningIcon } from '@motech-development/breeze-ui/icons';
```

## When to use

Use it to clarify a control whose visible label cannot carry a short secondary explanation. Use persistent text for essential instructions, `Popover` for interactive content, and `VisuallyHidden` to supply an accessible label that does not need a visual popup. Tooltips never replace a control's accessible name.

## Basic example

```tsx
import { Tooltip } from '@motech-development/breeze-ui';

<Tooltip.Root>
  <Tooltip.Trigger appearance="outline">Export</Tooltip.Trigger>
  <Tooltip.Content>Download the current results</Tooltip.Content>
</Tooltip.Root>;
```

## Anatomy

| Part                  | Purpose                                                               |
| --------------------- | --------------------------------------------------------------------- |
| `Tooltip.Root`        | Coordinates delayed hover, focus, touch, and controlled open state.   |
| `Tooltip.Trigger`     | Full public `Button` used as an action described by the tooltip.      |
| `Tooltip.IconTrigger` | Compact focusable informational icon with a required accessible name. |
| `Tooltip.Content`     | Concise, non-interactive descriptive popup.                           |

Use exactly one trigger part. `IconTrigger.children` should be a decorative icon and `aria-label` names the icon itself. The tooltip adds description; it does not supply that name.

```tsx
<Tooltip.Root>
  <Tooltip.IconTrigger aria-label="Warning information" variant="danger">
    <WarningIcon aria-hidden="true" />
  </Tooltip.IconTrigger>
  <Tooltip.Content variant="danger">
    This item requires attention
  </Tooltip.Content>
</Tooltip.Root>
```

## State, timing, and presentation

Use `defaultOpen` for uncontrolled state, `open` with `onOpenChange` for controlled state, or `open` with `readOnly` for immutable state. `onOpenChange` receives the complete next Boolean. Opening delay defaults to `700`ms and closing delay to `0`ms; use shorter delays only where repeated inspection warrants it.

Content placement defaults to `top`; supported values are `top`, `bottom`, `left`, `right`, and top/bottom start/end. Collision handling may change the final position. `offset` defaults to `6` pixels. `variant` is `secondary` by default and may be `primary` or `danger`; keep icon and content variants aligned and never use colour as the only meaning.

Trigger inherits Button disabled and loading behaviour. A disabled native button cannot receive focus or hover consistently, so essential disabled explanations should remain visible. Tooltip itself has no invalid, empty, error, or loading state and Content must not contain interactive elements.

## Native attributes and styling

IconTrigger and Content accept relevant `aria-*`, `data-*`, className, and element refs. Inline styling and conflicting interaction handlers are excluded.

## Keyboard, touch, internationalisation, and mistakes

Keyboard focus and pointer hover reveal Content after the delay; Escape dismisses it. Touch behaviour is provided by React Aria without making Content interactive. Keep content to a short phrase, translate it independently of stable application state, and allow wrapping. Start/end placement follows direction.

Do not put links, buttons, form controls, rich errors, or essential instructions in Content; do not use an unnamed IconTrigger; do not repeat the visible trigger label without adding information. Related components: `Popover`, `VisuallyHidden`, `IconButton`, and `Alert`.
