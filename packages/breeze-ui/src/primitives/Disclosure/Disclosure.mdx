# Disclosure

`Disclosure` reveals or hides one section of supporting content from a semantic heading button.

## Import

```tsx
import { Disclosure } from '@motech-development/breeze-ui';
```

## When to use

Use it for optional details attached to one context.

## When not to use

Use `Accordion` for several coordinated sections, `Dialog` for a modal task, and permanently visible content when hiding it would impede completion or discovery.

## Basic example

```tsx
import { Disclosure } from '@motech-development/breeze-ui';

<Disclosure.Root defaultOpen>
  <Disclosure.Trigger>Advanced details</Disclosure.Trigger>
  <Disclosure.Panel role="region">Additional information</Disclosure.Panel>
</Disclosure.Root>;
```

## Anatomy and state

| Part                 | Purpose                                          |
| -------------------- | ------------------------------------------------ |
| `Disclosure.Root`    | Coordinates expansion and disabled state.        |
| `Disclosure.Trigger` | Heading button that toggles the panel.           |
| `Disclosure.Panel`   | Collapsible content associated with the trigger. |

Use `defaultOpen` for uncontrolled state, `open` with `onOpenChange` for controlled state, or `open` with `readOnly` for immutable state. `onOpenChange` receives the complete next Boolean. `disabled` prevents the trigger changing state; an already open panel remains readable. There are no loading, invalid, empty, or error states.

`headingLevel` defaults to `3` and should follow document hierarchy. Panel `role` defaults to `'group'`; use `'region'` sparingly for significant sections.

## Native attributes and styling

Parts accept `className`, relevant HTML/ARIA attributes, and element refs. Inline styles and native click/change handlers are excluded where Breeze owns behaviour.

## Accessibility and common mistakes

Tab focuses the trigger; Enter or Space toggles it. Keep the trigger meaningful independently and do not place another interactive control inside it. Expansion state and trigger/panel association are supplied automatically. Translate the trigger without changing application state; wrapping and logical alignment follow direction.

Do not combine controlled and uncontrolled props, use a disclosure to conceal required form errors, or add a second panel under the same trigger. Related components: `Accordion`, `Dialog`, `Popover`, and `Typography`.
