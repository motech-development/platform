# Accordion

`Accordion` organises related sections into labelled panels whose expansion is coordinated by one root.

## Import

```tsx
import { Accordion } from '@motech-development/breeze-ui';
```

## When to use

Use it to reduce the initial length of supporting content while keeping several related headings visible. Use `Disclosure` for one independent section, `Tabs` when exactly one peer view should be active, and ordinary headings when all content should remain visible or searchable at once.

## Basic example

```tsx
import { Accordion } from '@motech-development/breeze-ui';

<Accordion.Root defaultValue={['delivery']}>
  <Accordion.Item id="delivery">
    <Accordion.Trigger>Delivery</Accordion.Trigger>
    <Accordion.Panel>Delivery details</Accordion.Panel>
  </Accordion.Item>
  <Accordion.Item id="returns">
    <Accordion.Trigger>Returns</Accordion.Trigger>
    <Accordion.Panel>Returns details</Accordion.Panel>
  </Accordion.Item>
</Accordion.Root>;
```

## Anatomy

| Part                | Purpose                                                       |
| ------------------- | ------------------------------------------------------------- |
| `Accordion.Root`    | Owns expansion keys and the single/multiple expansion policy. |
| `Accordion.Item`    | One stable keyed disclosure.                                  |
| `Accordion.Trigger` | Heading button that toggles its item.                         |
| `Accordion.Panel`   | Collapsible content associated with the trigger.              |

`Trigger` must precede its associated `Panel` inside each item. `headingLevel` defaults to `3`; choose the level that preserves the page heading hierarchy. A panel defaults to group semantics and may use `role="region"` when it is important enough to be a landmark.

## Expansion and states

`value` and `defaultValue` are arrays of complete `string | number` item ids. Use `defaultValue` for uncontrolled expansion, `value` with `onChange` for controlled expansion, or `value` with `readOnly` for immutable expansion. `onChange` receives the complete next array. `multiple` defaults to `false`; when true, items expand independently.

Root `disabled` disables every trigger; item `disabled` disables only one. Disabled open content remains readable. Accordion has no loading, invalid, empty, or error state: render feedback inside a panel, or use `StatePanel` instead of an empty accordion.

## Native attributes and styling

Parts accept `className`, relevant semantic attributes, and refs to their rendered elements. The Trigger supports relevant native button attributes; Breeze owns click, disabled, and inline style behaviour.

## Keyboard, accessibility, internationalisation, and mistakes

Tab visits each enabled trigger; Enter or Space toggles it. Focus does not rove with arrow keys. The trigger exposes expanded state and its panel relationship. Keep trigger text meaningful when heard alone, avoid putting controls inside it, and let translated headings wrap. Logical text alignment follows direction; ids remain stable.

Do not use array positions as ids, combine `value` with `defaultValue`, use `multiple` when content order implies a single choice, or hide essential primary content solely to shorten a page. Related components: `Disclosure`, `Tabs`, `Stepper`, and `NavigationList`.
