# SegmentedControl

`SegmentedControl` presents a short, mutually exclusive choice as adjacent selectable segments.

## Import

```tsx
import {
  SegmentedControl,
  type SegmentedControlOption,
} from '@motech-development/breeze-ui';
```

## When to use

Use `SegmentedControl` for two or a few concise peer choices that immediately change a view or presentation mode. Every option is visible and selection is represented by a stable string value.

Use `RadioGroup` for a labelled form question, required answer, native form participation, descriptions or errors. Use `ToggleGroup` when you need multiple selection or compound child composition. Use `Tabs` when selection switches between labelled content panels, and `ButtonGroup` for momentary actions.

## Basic example

```tsx
import { SegmentedControl } from '@motech-development/breeze-ui';
import { useState } from 'react';

const displayOptions = [
  { label: 'List', value: 'list' },
  { label: 'Grid', value: 'grid' },
];

export function DisplayMode() {
  const [mode, setMode] = useState('list');

  return (
    <SegmentedControl
      aria-label="Display mode"
      onChange={setMode}
      options={displayOptions}
      value={mode}
    />
  );
}
```

## Anatomy and options

SegmentedControl is a fixed pattern rather than a public compound namespace. The root coordinates one selection and keyboard focus. Each entry in `options` becomes one adjacent toggle with radio semantics in single-selection mode.

```ts
interface SegmentedControlOption {
  disabled?: boolean;
  label: ReactNode;
  value: string;
}
```

Values must be unique, stable, locale-independent strings. Translate `label`, not `value`. `disabled` on an option prevents that option from being selected; root `disabled` prevents all choices from changing.

Keep the set short and labels concise. The component has no loading, invalid, error or dedicated empty presentation. Avoid rendering an empty `options` array; render an application fallback instead. If options are loaded asynchronously, show separate loading and error feedback before rendering the control.

## Selection state and callback semantics

- **Uncontrolled:** omit `value`; optional `defaultValue` selects the initial option. Optional `onChange` observes subsequent non-empty selections.
- **Controlled:** provide `value` and `onChange`. The callback receives the next selected option’s string value, never an array or DOM event.
- **Read-only:** provide `value` with `readOnly`; focus is preserved but selection cannot change.

The public value is a single string. An initial empty selection is possible when neither `defaultValue` nor `value` is supplied. The underlying single-selection group can represent no selection, but `onChange` only reports a selected string and does not report an empty array. Use controlled state or `RadioGroup` when the application must enforce and validate a required answer.

Controlled and uncontrolled props are mutually exclusive. Mutable controlled state requires `onChange`; SegmentedControl throws if JavaScript bypasses that TypeScript contract. Application state owns persistence, URL synchronisation, data loading and consequences of the selected mode.

## Orientation, keyboard and direction

`orientation` defaults to `'horizontal'`. Horizontal segments form one row; vertical segments form one column. Orientation controls both layout and arrow-key axis.

- Tab enters or leaves the group. Enter or Space selects the focused segment.
- Horizontal arrow keys move focus according to provider text direction. Vertical orientation uses Up and Down.
- Selected options expose radio-style checked state. The required `aria-label` names the group; visible option labels name the choices.
- Keep DOM option order logical and do not reverse the array for right-to-left content.
- Option labels are application-owned, translated copy and must tolerate expansion. Values are identifiers and are not localised.

SegmentedControl performs no routing and is not a tab-panel coordinator. It has fixed default ToggleButton appearance and size; there are no pattern props for appearance, variant, size, density or tone. `className` is for placement, not internal segment styling.

## Common mistakes and related components

Do not use translated labels as values, include many or long options, use segments for unrelated actions, duplicate values, or treat the control as navigation tabs. Use `RadioGroup` for a form answer, `ToggleGroup` for configurable selection, and `Tabs` for content panels.
