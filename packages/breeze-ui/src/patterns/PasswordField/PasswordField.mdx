# PasswordField

`PasswordField` provides labelled password entry with supporting guidance, validation feedback and a show-or-hide action.

## Import

```tsx
import { PasswordField } from '@motech-development/breeze-ui';
```

## When to use

Use `PasswordField` for a current or new password when the canonical label, input, description, error and visibility composition is sufficient.

## When not to use

Use compound `TextField` with `type="password"` when you need native attributes or composition points that PasswordField does not expose. Use a normal `TextField` for non-secret text. PasswordField does not validate password strength, compare fields, generate passwords, store credentials or submit forms.

## Basic example

```tsx
import { PasswordField } from '@motech-development/breeze-ui';
import { useState } from 'react';

export function NewPasswordField() {
  const [password, setPassword] = useState('');

  return (
    <PasswordField
      autoComplete="new-password"
      description="Use at least 12 characters."
      label="New password"
      name="new-password"
      onChange={setPassword}
      required
      value={password}
    />
  );
}
```

## Anatomy

PasswordField is a fixed pattern rather than a public compound namespace.

| Region            | Responsibility                                                          |
| ----------------- | ----------------------------------------------------------------------- |
| Label             | Required persistent accessible name for the native input.               |
| Input group       | One visual surface containing the password input and visibility action. |
| Visibility action | Toggles the native input between `type="password"` and `type="text"`.   |
| Description       | Optional persistent requirements or guidance associated with the input. |
| Error             | Optional validation message associated while the field is invalid.      |

Visibility is internal presentation state and always starts hidden. It does not alter the password string. Read-only values may still be revealed; disabled fields disable both editing and the visibility action.

## Value state and callbacks

- **Uncontrolled:** omit `value`; `defaultValue` defaults to an empty string. Optional `onChange` observes edits.
- **Controlled:** provide `value` and `onChange`. The callback receives the complete next password string, never a DOM event.
- **Read-only:** provide `value` with `readOnly`; omit `onChange` and `defaultValue`.

Controlled and uncontrolled props are mutually exclusive. Mutable controlled state requires `onChange`; PasswordField throws if JavaScript bypasses that public TypeScript contract. The application owns password rules, strength calculations, matching, submission, server errors and security handling.

## Sizes, validation and browser behaviour

`size` accepts `'sm'`, `'md'` or `'lg'` and defaults to `'md'`; it sizes both the input and visibility action. Appearance and semantic colour are fixed.

Set `invalid` with meaningful `error` copy to expose invalid state and its associated message. `required` supplies native and accessible required state but does not invent a business rule. `description` remains visible for persistent requirements. `placeholder` is optional example text and never replaces `label`.

Use `autoComplete="current-password"` for an existing credential and `autoComplete="new-password"` for creation or reset. `'off'` is supported but browsers and password managers may choose their own security behaviour. `name` participates in native submission. There is no loading or empty-content mode; use the empty string as the empty value and keep pending submission state on the form action.

## Keyboard, accessibility and internationalisation

- The native input supports ordinary text-entry, selection, copy and paste keyboard behaviour. Tab moves to the visibility button; Enter or Space toggles visibility.
- The visibility icon is decorative. Its required accessible label switches between the provider messages `showPassword` and `hidePassword`; configure those messages for the application locale.
- Keep `label` visible and translate label, guidance and validation copy. Password values are opaque strings and are never translated or reformatted.
- Provider direction is inherited by the field. Do not alter or reverse a password based on writing direction.
- Disabling prevents focus and editing. Read-only preserves focus and copying; explain immutable state when it is not obvious.

PasswordField performs no routing. It exposes no custom class, ref, id, input mode, length constraints, pattern, ARIA attribute pass-through or custom visibility icon; compose `TextField`, `InputGroup` and `IconButton` directly when those deliberate public capabilities are needed.

## Common mistakes and related components

Do not put strength instructions only in a placeholder, use `invalid` without useful error copy, log password values in callbacks, disable password-manager support without a concrete reason, or assume the visibility toggle changes value state. Use `TextField` for custom password anatomy and `FormSection` or `FormActions` for surrounding form structure.
